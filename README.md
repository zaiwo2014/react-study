# React 前端工程化框架

**特点**
```text
1. 组件化 : UI 可以被拆成一个个可以复用的组件 -> 组件 = 函数 , 符合当下函数式编程的主流.
2. 单向数据流 : 数组只能父传子, 子组件不能直接改变父组件的状态.
3. State 驱动视图 : 响应式编程,不操作 DOM,只操作数据.
4. 虚拟 DOM: React 在内存中维护了一颗虚拟 DOM 的树, 使用 diff 算法对比新旧树,可以做到最小化真实 DOM 的操作,性能卓越.
5. React 只是单纯的视图层(view), 也就是把把数据通过 jsx 输出成 html 页面, 不管其他操作诸如(路由、状态管理、请求).
```


### 菜单如何与静态路由进行映射?
静态路由无所谓被看到,因为是否可以访问数据最终还是要通过后端权鉴,如果要根据用户角色进行菜单的显示隐藏, 也就是从后端获取菜单数据, 可以在菜单和路由的数据结构中维护一个 key 值, 从后端获取菜单数据后根据 key 值绑定对应路由, 实现半动态菜单, 即静态路由 + 动态菜单, 如果使用动态路由就太复杂了, 也没有必要, 在前端考虑系统安全是很愚蠢的行为.
